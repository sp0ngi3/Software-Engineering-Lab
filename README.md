# Software Engineering Lab

A personal learning repository for practicing software engineering fundamentals through small, focused experiments.

The purpose of this repository is to learn by building things from scratch, testing assumptions, debugging real code, and improving implementations over time. It is a workspace for understanding how things work under the surface.

## Current Focus

- Data structures and algorithms

## Projects

### DataStructures

Contains implementations of data structures written for learning purposes. The goal is to implement core behavior manually instead of relying on built-in equivalents.


Each exercise can include implementation notes, edge cases, complexity analysis, and tests.

### Playground

A console application used for manual experiments and debugging.



## Learning Approach

For each topic, the general process is:

1. Understand the concept before writing code.
2. Build a first version from scratch.
3. Debug it step by step.
4. Add tests for normal cases and edge cases.
5. Refactor after the behavior is understood.
6. Write down useful observations when needed.
